# Руководство по безопасному самоулучшению (Self-Improvement Guide)

Привет, Агент! Ты находишься в режиме самоулучшения. Твоя задача — помогать пользователю развивать, исправлять и оптимизировать этот веб-интерфейс (React + Vite + TypeScript).

Для защиты стабильности сервера и предотвращения синтаксических ошибок, в проект внедрена **автономная изолированная песочница сборки и автоматического исправления ошибок**.

---

## 🚫 ЧТО ДЕЛАТЬ НЕЛЬЗЯ:
* **НЕ записывай файлы напрямую в папку `src/`** с помощью обычного перезаписывания, если ты не уверен на 1000% в отсутствии синтаксических или импортных ошибок. Ошибка в импорте сломает билд и закроет доступ пользователям.

---

## ⚙️ КАК ДЕЙСТВОВАТЬ ПРАВИЛЬНО:

Всегда используй бэкенд-роуты песочницы для проверки сборки и безопасного деплоя. 

### 1. Безопасная проверка и Деплой (Рекомендуемый путь)
Отправь HTTP-запрос `POST /api/sandbox/apply` (для авторизации используй текущий заголовок авторизации сессии `X-Auth-Token` или куки).

**Параметры запроса (JSON):**
* `files` (Array): Список изменяемых файлов. Каждый элемент должен содержать `{ path: "относительный_путь", content: "весь_новый_код" }`.
* `dryRun` (Boolean): 
  * `true` (по умолчанию) — только проверить компиляцию в `/tmp` песочнице и вернуть ошибки, не меняя рабочий проект.
  * `false` — проверить компиляцию, отформатировать с помощью Prettier, и при успехе автоматически скопировать файлы в рабочий проект и сделать Git-чекпоинт.

#### 🔄 Авто-исправление в цикле (Auto-Correct Loop):
Если компиляция провалится, бэкенд в фоне **сам вызовет локальную языковую модель**, передаст ей лог ошибок TypeScript и попытается автоматически сгенерировать исправленный вариант. Тебе вернется финальный результат.

---

### 2. Точечные семантические изменения через AST (Abstract Syntax Tree)
Если тебе нужно точечно изменить файл (например, добавить импорт или вшить новый API-роут на сервере) и ты боишься сломать структуру фигурных скобок, используй `POST /api/sandbox/ast-modify`.

**Параметры запроса (JSON):**
* `filePath` (String): Путь к файлу (например, `"server/index.mjs"` или `"src/components/SettingsPanel.tsx"`).
* `dryRun` (Boolean): `true` для проверки, `false` для записи.
* `operations` (Array): Список операций:
  * **Добавить импорт (`addImport`):**
    ```json
    { "action": "addImport", "module": "path/or/npm", "specifiers": ["NamedExport"], "default": "DefaultExport" }
    ```
  * **Добавить API-роут на сервере (`addRoute`):**
    ```json
    { "action": "addRoute", "method": "GET", "route": "/api/my-new-endpoint", "handlerCode": "res.writeHead(200); res.end(JSON.stringify({ ok: true }));" }
    ```

---

## ⚡ ПОСЛЕ УСПЕШНОГО ДЕПЛОЯ:
После того как песочница вернула `"status": "success"`, файлы на диске обновлены. Чтобы пользователь увидел изменения:
1. Вызови `POST /api/rebuild`, чтобы пересобрать клиентский бандл Vite.
2. Логи сборки и все подробности ты и пользователь можете увидеть в **Консоли событий** в настройках UI.
3. Сообщи пользователю, что код обновлен и компилируется, и предложи обновить страницу!
