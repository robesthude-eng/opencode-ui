# 🎯 Согласованная архитектурная спецификация: Режим Самоулучшения (Self-Improvement)

Этот документ фиксирует финальную, полностью согласованную с вами техническую спецификацию для реализации автоматического режима саморазвития в проекте **OpenCode UI**.

---

## 🗺️ Итоговая архитектура сквозного цикла

```text

| Администратор    | ---------------------> | Чат ИИ + Очистка старых песочниц         |
+------------------+                        +-----------------------------------------+
                                                                 |
                                                                 v
+------------------+     Отображение в UI   +-----------------------------------------+
| Панель Workspace | <--------------------- | Копирование чистых файлов из Docker-src |
+------------------+                        +-----------------------------------------+
         |                                                       |
         | Пишет ТЗ                                              v
         |                                  +-----------------------------------------+
         +--------------------------------> | ИИ правит файлы в Sandbox (гибкий)     |
                                            +-----------------------------------------+
                                                                 |
                                                                 v
+------------------+    Статус на экране    +-----------------------------------------+
| Индикатор тестов | <--------------------- | Сервер сам гонит npm run test после прав|
| (Светофор)       |                        +-----------------------------------------+
         |                                                       |
         | Если тесты зеленые                                    v
         |                                  +-----------------------------------------+
```

## 📋 Подробное описание каждого шага

### ✦ Шаг 1. Инициализация чата и песочницы (Sandbox)
* **Пользовательский опыт:** Администратор нажимает на боковой панели кнопку **«✦ Сессия саморазвития»**. Чат инициализируется мгновенно.
* **Приветствие ИИ:** Ассистент сразу отправляет в чат приветственное сообщение, описывая структуру файлов в панели Workspace и заявляя о полной готовности к работе.
* **Безупречная зачистка (Teardown & Setup):**
  * Любой мусор от старых песочниц полностью стирается.
  * Песочница `/app/workspace/sessions/{sessionId}/workspace` создается с нуля.
  * Исходные файлы копируются из **чистого эталонного Docker-образа (`/app/workspace-src`)**, что гарантирует отсутствие накопленных багов и абсолютную чистоту исходников при старте.
* **Изолированные загрузки (Uploads):** Любые загружаемые пользователем файлы во время активного сеанса саморазвития попадают **строго в папку песочницы сессии** (`sessions/{id}/workspace/uploads`), чтобы ИИ видел их локально в воркспейсе, а живой сервер оставался чистым от мусора.

---

### ✦ Шаг 2. Разработка и автоматическое тестирование
* **Гибкий доступ к кодовой базе:** Ассистент имеет гибкий доступ к коду песочницы (может править `src/`, `public/`, `package.json`, `vite.config.ts`), что дает ему возможность проводить глубокий рефакторинг и подключать новые библиотеки из npm.
* **Автоматические локальные тесты (Continuous Integration):**
  * Администратору и ассистенту больше не нужно вручную запускать тесты.
  * Сервер автоматически запускает `npm run test` после каждого сохранения файла ИИ-агентом.
  * **Визуальный Светофор:** Результаты тестов отображаются в реальном времени в шапке чата в виде лаконичного индикатора-светофора (`🟡 Идет тест...` / `🟢 Тесты пройдены` / `🔴 Тесты провалены`).

---

### ✦ Шаг 3 & 4. Создание PR, авто-слияние и петля авто-исправления
* **Авто-слияние на GitHub (Auto-Merge на Green CI):**
  * Если тесты в песочнице зеленые, ассистент создает коммит, пушит ветку `si/*` и открывает PR на GitHub.
  * Если тесты GitHub Actions CI завершаются успешно, PR автоматически сливается (auto-merge), запускается деплой на VDS Timeweb, а ИИ-агент пишет отчет в чат о проделанной работе.
* **Автономная петля самоисправления (Auto-Fix Loop на Failed CI):**
  * Если тесты на GitHub завалились, фоновый воркер `startPrStatusPoller` перехватывает лог ошибок, форматирует его, экранирует от инъекций и автоматически отправляет обратно в чат.
  * Ассистент автоматически приступает к исправлению ошибок, правит код, коммитит и пушит фикс.
  * **Защитный предохранитель:** ИИ разрешено сделать максимум **3 автоматические попытки исправления** подряд. Если после этого CI продолжает падать, цикл приостанавливается, и воркер запрашивает ручную помощь администратора, чтобы уберечь лимиты токенов от бесконечного выжигания.

---

### ✦ Шаг 5. Аварийный откат и Резервное копирование
* **Резервная копия ПЕРЕД каждым деплоем:**
  * Чтобы исключить малейший риск повреждения данных, сервер автоматически создает резервную копию базы данных SQLite прямо **перед каждым применением изменений** на живом сервере.
* **Цветовая связь по деплоям (Color-coded Sync):**
  * Каждая сборка UI и бэкап БД, созданные в рамках одного деплоя, подсвечиваются в панели управления общим цветовым тегом/маркером. Это позволяет сразу визуально определить их временное соответствие.
* **Выборочный ручной откат (Selective Rollback):**
  * В панели настроек `SettingsPanel` администратор получает полный контроль. Вы можете откатить только интерфейс (UI), только базу данных (БД) к определенному снимку, либо оба элемента одновременно и независимо.
